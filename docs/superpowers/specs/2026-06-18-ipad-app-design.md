# ouchi-master iPadアプリ 設計書

作成日: 2026-06-18

---

## 1. サービス概要

子どもの日々の家事の手伝いや生活習慣の定着を、家庭内でポジティブに促進・管理するデジタルポイントシステム。
既存のアナログ（紙・手書き）運用からのデジタルリプレイスを目的とし、MVPからリリースして家族の実際の反応をベースに反復改善する。

### 家族構成
- 子ども: はる、あきら（2名）
- 運用想定: リビングのiPadに常時表示

---

## 2. 技術スタック選定

| レイヤー | 技術 | 選定理由 |
|---------|------|---------|
| フレームワーク | Flutter / Dart | 後述 |
| 状態管理 | Riverpod | Flutterのデファクトスタンダード。ホットリロードとの相性◎ |
| ローカルDB | Drift（SQLite） | OS標準搭載のSQLiteを型安全なDartで扱える。狙い通りのソリューション |

### Flutter選定理由
- **ホットリロードによる超高速な仮説検証**: 家族のフィードバックをその場でiPad実機に反映し、MVPの改善サイクルを最速で回せる
- **表現力の高いリアルタイムUI演出**: 独自の描画エンジンにより、iPadの大画面を活かした滑らかなアニメーションやリアルタイムなポイント加算演出を自由に実装できる
- **将来的なマルチデバイス展開の容易さ**: 単一コードベースからiOS/Android/Webに出力できるため、将来的な親の入力用Web画面やスマホアプリへの拡張コストを最小化できる
- **要件特性によるデメリットの無力化**: クローズドな家庭内システムのため、Flutter Webの弱点（SEO・初期読み込み）を完全に無視できる
- **低い学習コストと高い保守性**: JavaやTypeScriptの経験があれば馴染める静的型付け・Null Safetyにより、個人開発でもバグを抑えて堅牢に書ける

---

## 3. アーキテクチャ方針

- **シングルデバイス**: iPadの1台運用。リアルタイムバックエンド不要
- **ローカルファースト**: 全データはiPad内のSQLiteに保存
- **データ削除なし**: 記録はソフトデリート（`deleted_at`）で管理。物理削除しない

### アニメーション演出

**加点時:**
1. 数字カウントアップ（0.6秒、ぬるっと増える）
2. 紙吹雪パーティクルエフェクト（該当子どものパネルから）
3. トースト通知（例:「はる ＋30P おさらあらい 🎉」）

**減点時:**
1. 数字カウントダウン（赤色）
2. トースト通知のみ（パーティクルなし。叱責感を出さない）

**効果音:** デフォルトOFF。設定でON可能（夜間・図書館考慮）

### タイムスタンプ形式

`6/18 18時`（月/日 時）。年は当月内なら省略。

## 7. データモデル

### テーブル設計

```
children
  id           INTEGER PRIMARY KEY
  name         TEXT
  color        TEXT
  created_at   DATETIME

categories
  id           INTEGER PRIMARY KEY
  name         TEXT
  emoji        TEXT
  sort_order   INTEGER

chore_templates
  id           INTEGER PRIMARY KEY
  category_id  INTEGER (FK → categories)
  name         TEXT
  points       INTEGER  ※ げんてんはマイナス値
  is_active    BOOLEAN
  sort_order   INTEGER

activity_logs
  id           INTEGER PRIMARY KEY
  child_id     INTEGER (FK → children)
  chore_name   TEXT    ※ 記録時点のスナップショット
  points       INTEGER ※ 記録時点のスナップショット
  recorded_at  DATETIME
  deleted_at   DATETIME (nullable) ※ ソフトデリート
```

### 設計の重要な判断

- **スナップショット保存**: `activity_logs`の`chore_name`と`points`は記録時点の値をコピーして保存。テンプレートを後から変更しても過去の記録が壊れない
- **ソフトデリート**: 誤入力時に`deleted_at`をセットして論理削除。物理削除しないため復元可能
- **月次リセットはUI概念**: データは削除せず`recorded_at`で月フィルタするだけ。過去の月の履歴はいつでも参照可能

### 月次集計クエリ

```dart
// 当月ポイント合計
SELECT SUM(points) FROM activity_logs
WHERE child_id = ?
  AND recorded_at >= 今月1日 00:00:00
  AND deleted_at IS NULL
```
